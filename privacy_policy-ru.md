
# Политика конфиденциальности

Исходные коды приложения открыты и доступны в репозитории для ознакомления
и анализа по адресу "https://github.com/maxrys/js-blocker".


## Структура приложения

Приложение является связкой из двух частей:
- главное приложение `JS Blocker`;
- Safari-расширение `JS Blocker Extension`.

Главное приложение состоит из:
- главного окна `JS Blocker` (в котором пользователь управляет списком
  доменов, на которых разрешена работа JavaScript: удаление, импорт, экспорт).

Safari-расширение состоит из:
- Всплывающее окно (в котором пользователь может добавить или удалить
  правило для текущего домена). Окно появляется при нажатии на
  кнопку `JS` на панели инструментов Safari.
- Локальный файл `script.js`. Основной назначение – выключение JavaScript
  на выбранном домене (по умолчанию) или его включение по требованию пользователя.
  Является прозрачным для анализа и необходим для функционирования приложения.

Запуск расширения `JS Blocker Extension` возможен лишь на одобренных
пользователем доменах (через настройки Safari).


## Модификация файловой системы

Приложение не работает с файловой системой – не записывает
и не читает никакие данные.


## Хранение данных

Приложение использует стандартный механизм `Core Data` для хранения "списка
разрешённых доменов" в локальной базе данных.

Данный список формируется по требованию пользователя для
функционирования приложения.

При включении опции "Экспериментальный / Включить CloudKit" происходит
синхронизация с iCloud.


## Другие активности

Приложение:
- работает в изолированной среде ОС (песочнице);
- не регистрирует нажатия пользователя;
- не получает доступ к буферу обмена;
- не добавляет события к DOM-элементам;
- не отправляет и не принимает данные по сети;
- не собирает и не хранит персональные данные;
- не передаёт данные третьей стороне;
- не подключает внешние файлы JavaScript.
